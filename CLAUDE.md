# TheGrowthPMM — marketing site

Single-page marketing site for **Divya Sadanandan** ("TheGrowthPMM"), an independent **product marketing / growth consultant**. Built with **Astro + Tailwind v4**, but most styling is hand-authored inline styles + a custom design system in `src/styles/global.css` (CSS variables, not Tailwind utilities).

## Run / verify
- Dev server: `npm run dev` (Astro, default port 4321).
- Prefer the `preview_*` tools to verify visually. Note: in past sessions, deep-scroll screenshots sometimes came back blank — verify layout via DOM (`getBoundingClientRect`, computed styles) when screenshots are unreliable, and reserve screenshots for the hero/top of page.

## Key files
- `src/pages/index.astro` — the whole homepage (nav, hero, logo strip, problem, services, how-it-works, proof, testimonials, why-me comparison, about, the free report as a soft section, closing CTA, footer).
- `src/pages/faq.astro` — separate FAQ page at `/faq` (six Q&As in a `<details>` accordion), linked from footer + mobile menu. Carries `FAQPage` JSON-LD.
- `src/pages/ai-product-marketing.astro` — AEO / thought-leadership pillar at `/ai-product-marketing` (AI PMM point of view, the workflows, guardrails, proof, a 4-Q FAQ). The "leading voice of AI PMM" content home. Linked from footer + mobile menu. Carries `FAQPage` JSON-LD.
- `src/styles/global.css` — design system: navy palette CSS vars, `.handwritten` (Caveat accents + `.light` variant), `.sticky-tag`, `.module` (Services cards), `.channel-grid`, `.compare-*` (Why-me: alternatives get ✗, `.compare-card--me` gets ✓), `.process-*` (How it works steps), `.testimonial-lead` + `.testimonial-pair` (editorial testimonials: a featured pull-quote with an oversized Caveat quote mark, then two borderless recs under a hairline — not cards, changed 2026-06-29), `.steps-flow` + `.report-checklist` (report card), `.funnel-*`, responsive rules, `.marker` (lime highlighter, in active use — see Design system below). (`.service-row`, `.audit-step` linger but are unused.)
- `src/layouts/Layout.astro` — `<head>`, fonts (Inter + Caveat), meta/OG, and the global **JSON-LD entity graph** (`Person` Divya Sadanandan + `ProfessionalService` TheGrowthPMM) emitted on every page so answer engines resolve the entity.

## Design system
- **LIME REBRAND (2026-07-01) — the palette is now lime + near-black, NOT navy. Don't revert to navy.** `--navy` is now the lime pop `#DBFF00` (the token kept its name to avoid renaming ~everywhere; it's a misnomer). Lime is only ever used as a **fill** (buttons, badge chips, the featured-card border) with **dark `--ink` text on top**, or as an **accent on the dark surfaces** (`.handwritten.light` cursive, the closing-band CTA). Lime is **never** text on a light background (unreadable, ~1.1:1). The dark surfaces that used to be navy (footer, closing band, Growth Audit card, recognition band, `.compare-card--me`) now use the new **`--ink-deep` #16180f** near-black. Accent text on light sections (headings' sublines, the big proof numbers, eyebrows, `.handwritten`) is dark `--ink`/`--ink-deep`, not lime. `.btn-navy`/`#navbar .btn-navy` text is `--ink` (was #fff); `::selection`, focus rings and the hamburger icon are dark, not lime. The `<Logo>` mark and `favicon.svg` are now a **lime `#DBFF00` rounded-square badge** with a near-black `#16180f` funnel + coral `#cc4117` dot (recolored 2026-07-01; the mark used to be a bare dark funnel — this brings the lime fill into the wordmark). The Logo `reverse` variant now only swaps the wordmark colour to white; the badge stays lime on both light nav and dark footer. See `HANDOFF.md` 2026-07-01 for the full token map.
- **Backgrounds:** cool navy-tinted neutrals (`--cream` #e7ebf5, `--blue-tint`, white; warm beige retired 2026-06-29). `--warm-line` #d6dcec is the cool hairline on tinted sections.
- **Type (updated 2026-07-01):** **Bricolage Grotesque** is the display face for all headings via `--font-display` (`h1,h2,h3`); **Inter** stays for body/UI; **Caveat** (`.handwritten`) for playful scrapbook accents. Coral (`#cc4117` logo dot, `#ec6a43` funnel drop-off balls) is the one supporting accent, meaning "drop-off". So the system is **lime + near-black + coral**.
- **Sticky tags are now the lime/near-black pair (2026-07-01):** `.sticky-tag.purple` = lime fill + dark text, `.sticky-tag.gold` = near-black fill + lime text (class names kept, hues retired). On the near-black recognition band the tag uses `.purple` (lime) so it stays visible.
- Playful elements (handwritten accents, rotated `.sticky-tag` notes, grid-canvas) are intentional. The **handwritten kicker was thinned 2026-07-01** — it no longer sits above nearly every section (dropped on Services/Proof/Testimonials; kept on How-it-works, About, right-fit, Report, closing). The `.marker` highlighter is **now lime and in use** on "next growth stage" (hero H1), "the numbers" (proof H2), and "only three clients a month" (About bio) to carry the brand colour through the light sections.
- The hero visual is a **static positioning diagram** (`.diagram-wrap`, inline SVG, replaced the funnel 2026-07-02 on a positioning-route change — see `HANDOFF.md`). It's a **hub-and-spoke**: three transparent outline chips (**Product** top, **Sales** + **Marketing** bottom) with thin near-black connectors converging into a central **lime "The Growth PMM" badge** (rotated -2°, the hub). A **Caveat handwritten "the missing piece"** annotation with a **curly hand-drawn arrow** points at the badge (the arrow was recut 2026-07-02 to sit in the open wedge above-right of the badge so it no longer overlaps the pill text or the annotation); a matching Caveat caption below reads "Between product-market fit and scalable growth." **No white card** (floats on the hero bg over the grid-canvas), **no coral** (nothing here represents drop-off anymore, so the hero is pure lime + near-black). It **teaches the positioning** (PMM is the missing leadership function connecting Product/Sales/Marketing after PMF) rather than illustrating a funnel. The old animated 3D funnel SVG + its `prefers-reduced-motion` SMIL-pause script were removed (static, nothing to pause).
- **Mobile:** the floating corner CTA (`#nav-cta-corner`) **docks to the bottom** (`bottom:16px`) below 900px — thumb zone, and it no longer overlaps section headings scrolling under the top-right corner (fixed 2026-07-01).
- **Icons are monoline SVGs, not emoji (2026-07-02, per critique):** Services (4), Recognition band (3, lime stroke on the dark band), and the `/ai-product-marketing` workflow cards (4) all use inline monoline `<svg>` (viewBox 0 0 24 24, `stroke-width:1.8`, ink-deep on light / lime on dark), each `aria-hidden`. No emoji anywhere as UI icons. The hero diagram SVG keeps `role="img"` + `<title>` (it conveys meaning, so it's NOT aria-hidden).
- **Proof stat labels are sentence case (2026-07-02):** the four category labels above the big numbers + "Also on record" dropped `text-transform:uppercase` + tracking (they were the tiny-uppercase-tracked eyebrow tell above metrics). Footer column headers, "Past experience", and the Growth-Audit card label stay uppercase (conventional utility labels in different contexts).
- **Secondary pages brought in line 2026-07-02:** `/ai-product-marketing` had drifted (uppercase `.eyebrow` on every section, emoji icons, "Book a call") — now uses handwritten Caveat kickers, monoline SVG icons, and "Book free consultation". `/faq` CTA also unified. `.eyebrow` (uppercase) CSS still exists but is no longer used on any page.

Mobile (`<900px`): 168px, left-aligned. It closes with an **"Is this the right fit?"** qualifier block (replaced the full-time-vs-Growth-PMM comparison, 2026-06-30; **layout revisited 2026-07-01** from a narrow centered island to a **left-aligned** block matching the site's section rhythm — a "let's be honest" handwritten eyebrow into the heading, then a 2-col `.compare-grid` at `max-width:900px`, cards ~442px each).
